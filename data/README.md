# Data Description

## Overview
This directory contains all input, intermediate, and processed data used in the DNS dependency analysis workflow. The structure separates immutable inputs, generated intermediates, and final analysis outputs to support reproducibility and clarity.

Large intermediate files are not version-controlled and must be generated locally by running the data-fetch pipeline.

---

## Directory Structure

### `data/names/`
Reference datasets used during preprocessing.

- `categorized_names.csv`  
  Mapping of domain names to IANA TLD categories (e.g., gTLD, ccTLD, sponTLD, infTLD).  
  Used to group domains for dependency analysis.

---


### `data/result/`
Main output directory for generated data.

#### `data/result/tmp/`  *(not version-controlled)*
Temporary intermediate files created during large-scale data fetching.

- Contains chunked outputs produced during dependency reconstruction.
- Used to avoid memory overload and enable resumable execution.
- These files are required for full reproducibility but are **not stored in the repository**.
- Must be generated by running:ppt-iyp-cisco-top1m-fetch
